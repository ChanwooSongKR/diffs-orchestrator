# 인터랙션 모델 결정문서

> 이 문서는 UI/UX 설계와 에이전트 오케스트레이션의 기준이 되는 세 가지 핵심 결정을 기록합니다.
> 이 결정들이 바뀌기 전까지, 모든 UI 컴포넌트와 에이전트 동작은 이 문서를 따릅니다.

---

## 결정 1. 인터랙션 모델 — **채팅 우선 (Chat-first)**

### 결정
사용자의 주 입력 수단은 자유형 채팅이다.
단계를 직접 클릭하거나 버튼을 눌러 다음으로 넘어가는 방식은 사용하지 않는다.

### 의미
- 사용자는 언제든 "더 도전적으로", "이 두 개 합쳐줘", "컷을 4개로 줄여줘"라고 타이핑한다.
- 오케스트레이터가 그 발화를 해석해 적절한 에이전트를 호출한다.
- UI는 "지금 어느 단계인지"를 명시적으로 강요하지 않는다.

### 단, 각 단계의 산출물은 구조화된 형태로 피드에 등장한다
- 드래프트는 카드로 렌더링된다
- 시나리오는 컷별로 구조화되어 보인다
- 콘티는 썸네일 스트립으로 보인다
- 사용자는 이를 보면서 채팅으로 개입한다

### 금지
- 탭, 사이드바 단계 클릭, "다음 단계" 버튼으로 플로우를 강제하는 것
- 사용자가 채팅 없이 버튼만으로 작업을 완료할 수 있게 만드는 것

## 결정 2. 에이전트 투명성 — **한 줄 상태 표시**

### 결정
에이전트가 실행 중일 때, 피드에 한 줄짜리 상태 메시지만 표시한다.
전체 에이전트 대시보드나 실시간 로그는 노출하지 않는다.

### 형식
```
[에이전트 이름] — [지금 하고 있는 일 한 줄]
```
예시:
- `리서치 에이전트 — 경쟁사 광고 캠페인 분석 중...`
- `Copy Draft Workers — 감성·직설·유머 방향 초안 병렬 생성 중...`
- `Shot Planning Agent — 시나리오를 5컷으로 분해하는 중...`

### 완료 시
한 줄 상태가 결과물 카드로 교체된다. 별도 "완료" 알림은 없다.

### 이유
- 병렬로 수십 개의 에이전트가 동시에 돌아도 UX가 복잡해지지 않는다.
- 사용자는 "시스템이 뭔가 하고 있다"는 신뢰를 갖되, 세부 작동에 압도되지 않는다.
- 더 자세한 로그가 필요하면 별도 "에이전트 로그" 패널(옵션)로 분리한다.

## 결정 3. 자동화 범위 — **게이트 기반 명시적 승인 + Agentic 오케스트레이션**

### 결정
사람의 명시적 승인이 필요한 **게이트(Gate)** 를 사전에 정의한다.
게이트 사이의 에이전트 호출은 오케스트레이터가 자율 판단한다.

### 승인 게이트 (총 4개)

```
사용자 발화 →
  [자동] 기획 정보 정리 + Memory 조회 + 리서치 + 레퍼런스 검색
      ↓
[GATE 1] 드래프트 선택
  사용자가 하나 이상의 드래프트를 선택하거나 "이 방향으로 가자"고 말해야 통과.
      ↓
  [자동] 컨셉 확정 + 필요 시 리믹스
      ↓
[GATE 2] 컨셉 승인
  사용자가 "이 컨셉으로 가자" 또는 수정 요청을 해야 통과.
      ↓
  [자동] 시나리오 작성 + 컷 분해 + 이미지 프롬프트 생성
      ↓
[GATE 3] 시나리오/컷 승인
  사용자가 컷 구성을 확인하고 승인해야 이미지 생성으로 진행.
      ↓
  [자동] 이미지 생성 + 콘티 구성 + QA 검수
      ↓
[GATE 4] 최종 승인
  사용자가 콘티를 확인하고 "완료" 또는 수정 요청.
```

### 게이트 사이에서 오케스트레이터가 자율 판단하는 것들
- 어떤 에이전트를 호출할지
- 병렬로 실행할지 순차로 실행할지
- 리서치가 부족하면 자동으로 추가 검색을 할지
- 레퍼런스 DB에서 몇 개를 가져올지

### 게이트 UI 패턴
게이트 도달 시 피드 하단에 **Approve Bar**가 등장한다:
```
[산출물 요약] — [수정 요청] [승인 →]
```
- "승인 →" 누르면 다음 자동 실행 구간 시작
- "수정 요청" 누르거나 채팅에 타이핑하면 해당 게이트에서 재작업
- 게이트는 **항상 피드의 가장 아래**에 위치한다 (스크롤로 지나칠 수 없음)

### 이유
- 사용자가 "AI가 알아서 다 했네"라고 느끼지 않도록 책임 소재를 명확히 한다.
- 4개의 게이트는 광고 제작 실무에서 실제로 의사결정이 필요한 시점과 일치한다.
- 게이트 사이는 자동화해서 사용자의 반복 클릭 피로를 제거한다.

## 이 결정들이 바뀌면 영향받는 것들

| 결정 변경 | 영향 범위 |
|-----------|-----------|
| 채팅 우선 → 단계 우선 | 전체 레이아웃 재설계, 오케스트레이터 라우팅 로직 |
| 한 줄 상태 → 대시보드 | Right Panel 완전 재설계 |
| 게이트 수 변경 | 오케스트레이터 상태 머신, UI Approve Bar 위치 |

## 미결 사항 — 결정 완료

| 항목 | 결정 | 비고 |
|------|------|------|
| 게이트 타임아웃 | **없음** — 세션 무기한 유지 | 광고 기획은 며칠에 걸쳐 진행되므로 강제 만료 없음 |
| 이전 게이트 롤백 | **가능** — 채팅으로 요청 | "드래프트 선택으로 돌아가고 싶어"라고 타이핑하면 오케스트레이터가 해당 게이트로 복귀. 이후 산출물은 새로 생성, 이전 것은 Diff 이력에 보존. |
| 협업 모드 (다중 사용자) | **v2 범위** — MVP 제외 | MVP는 1인 세션 기준 설계 |
| Memory 직접 편집 | **v2 범위** — MVP 제외 | MVP에서 Memory는 읽기 전용. 사용자는 채팅으로 간접 수정 가능 ("이 브랜드는 유머 톤 금지야" 등) |
