# 대화형 광고 시나리오 AI 멀티에이전트 시스템 요구사항

## 1. 문서 목적

본 문서는 사용자와의 대화를 기반으로 광고 아이디어를 탐색하고, 최종적으로 광고 시나리오, 컷 구성, 이미지 생성 프롬프트, 콘티 시각화까지 생성하는 AI 멀티에이전트 시스템의 요구사항을 정의한다.

이 시스템은 단일 텍스트 생성기가 아니라, 광고 기획 수집부터 리서치, 카피 탐색, 시나리오 작성, 컷 분해, 이미지 프롬프트 생성, 콘티 구성, 수정 이력 축적까지의 전체 작업 흐름을 지원해야 한다.

## 2. 시스템 목표

시스템은 다음 목표를 만족해야 한다.

1. 사용자와 대화하며 광고 작성을 위한 기획 정보를 점진적으로 수집할 수 있어야 한다.
2. 브랜드, 제품, 시장, 타겟, 경쟁 상황에 대한 정보가 부족할 경우 내부/외부 리서치를 수행할 수 있어야 한다.
3. 내부 레퍼런스 DB와 외부 검색 결과를 함께 활용하여 참고 가능한 광고 사례와 차용 요소를 도출할 수 있어야 한다.
4. 다양한 카피 및 컨셉 드래프트를 병렬로 생성하고, 사용자가 이를 선택, 비교, 조합, 재생성할 수 있어야 한다.
5. 확정된 컨셉을 기반으로 광고 시나리오를 작성할 수 있어야 한다.
6. 시나리오를 컷 단위로 분해하고 각 컷의 목적, 구도, 시간 배분을 정의할 수 있어야 한다.
7. 각 컷에 대해 이미지 생성 모델에 바로 활용 가능한 수준의 프롬프트를 생성할 수 있어야 한다.
8. 생성된 이미지와 컷 정보를 바탕으로 검토 가능한 콘티를 구성할 수 있어야 한다.
9. 전체 과정에서 발생한 선택, 폐기, 수정 패턴을 diff 형태로 누적 저장할 수 있어야 한다.

## 3. 핵심 원칙

### 3.1 대화형 워크플로우

사용자는 처음부터 완성된 브리프를 제공하지 않을 수 있다. 시스템은 대화를 통해 부족한 정보를 보완하고, 불확실한 항목과 확정된 항목을 구분해 관리해야 한다.

### 3.2 카피 탐색 중심 구조

시스템은 바로 시나리오를 생성하지 않고, 먼저 다수의 카피/컨셉 드래프트를 생성한 뒤 사용자가 선택, 비교, 조합, 재생성하는 과정을 핵심 워크플로우로 가져야 한다.

### 3.3 병렬 처리 기반 구조

시스템은 다음 작업을 병렬 처리 가능하도록 설계해야 한다.

1. 기획 정보 정리
2. Memory 조회
3. 내부 레퍼런스 검색
4. 외부 웹 리서치
5. 카피 드래프트 다중 생성
6. 선택안 기반 리믹스 생성
7. 컷별 이미지 프롬프트 생성
8. 컷별 이미지 생성
9. 콘티 시각화 버전 다중 생성

### 3.4 메모리 및 diff 축적

시스템은 감독별 취향, 프로젝트별 방향성, 선택된 안과 폐기된 안, 반복 수정 패턴을 지속적으로 저장하고 이후 생성 과정에서 재활용할 수 있어야 한다.

### 3.5 레퍼런스 활용 방식

레퍼런스는 원본 복제 목적이 아니라 차용 가능한 구조적 요소, 톤, 감정선, 포맷, 연출 방식 등을 추출하기 위한 참고 자료로 활용되어야 한다.

### 3.6 리서치 기반 생성

광고 생성은 브랜드 및 제품 이해를 기반으로 해야 한다. 정보가 부족할 경우 시스템은 검색과 리서치를 통해 맥락을 보강해야 하며, 리서치 결과는 단순 참고가 아니라 이후 카피 생성, 시나리오 작성, 레퍼런스 탐색의 공통 입력으로 재사용되어야 한다.

### 3.7 인터랙션 모델 — 채팅 우선 (Chat-first)

사용자의 주 입력 수단은 자유형 채팅이다. 단계 버튼 클릭이나 탭 전환으로 플로우를 강제하지 않는다.

- 사용자는 언제든 자연어로 요청한다. ("더 도전적으로", "A와 B를 합쳐줘", "컷을 4개로 줄여줘")
- Conversation Manager Agent가 발화를 해석하고, 오케스트레이터가 필요한 에이전트를 자율 판단하여 호출한다.
- 각 에이전트의 산출물(드래프트, 시나리오, 콘티 등)은 구조화된 형태로 피드에 등장한다.
- 에이전트 실행 중에는 **한 줄 상태 메시지**만 표시한다. (`[에이전트 이름] — [지금 하고 있는 일]`)

### 3.8 게이트 기반 명시적 승인

전체 흐름에는 사람의 명시적 승인이 필요한 **게이트(Gate)** 가 4개 존재한다. 게이트 사이의 에이전트 호출은 오케스트레이터가 자율 결정한다.

```
[자동] 기획 수집 + 리서치 + 레퍼런스 검색
    ↓
[GATE 1] 드래프트 선택
    사용자가 방향을 선택하거나 채팅으로 지시해야 통과.
    ↓
[자동] 컨셉 확정 (+ 필요 시 리믹스)
    ↓
[GATE 2] 컨셉 승인
    사용자가 컨셉을 확인하고 진행 의사를 밝혀야 통과.
    ↓
[자동] 시나리오 작성 + 컷 분해 + 이미지 프롬프트 생성
    ↓
[GATE 3] 시나리오/컷 승인
    사용자가 컷 구성을 확인해야 이미지 생성으로 진행.
    ↓
[자동] 이미지 생성 + 콘티 구성 + QA 검수
    ↓
[GATE 4] 최종 승인
    사용자가 콘티를 확인하고 완료 또는 수정 요청.
```

**게이트 동작 규칙:**
- 게이트 타임아웃 없음. 사용자가 돌아올 때까지 세션 상태를 유지한다.
- 채팅으로 이전 게이트 복귀 가능. ("드래프트 선택으로 돌아가고 싶어") 복귀 후 산출물은 새로 생성되고, 이전 것은 Diff 이력에 보존된다.
- 게이트 UI는 **Approve Bar** 형태로 피드 하단에 고정 표시된다.

2. Task Router / Scheduler
3. Result Aggregator

### 4.2 컨텍스트 및 리서치 레이어

3. Diff Tracker Agent
4. Internal Reference DB Search Agent

1. Copy Draft Generator Workers
2. Copy Curator Agent

2. Shot Planning Agent

1. Storyboard Composition Agent

### 4.6 검수 레이어

1. Review / QA Agent
2. Brand / Policy Checker

## 5. 전체 처리 흐름

### 5.1 단계 1: 사용자 요청 수신

Conversation Manager Agent는 사용자 입력을 수신하고 현재 세션 상태를 확인해야 한다. 초기 단계에서는 다음 작업을 병렬 수행할 수 있어야 한다.

1. 기획 정보 정리
2. Memory 조회
3. 내부 레퍼런스 검색
4. 외부 리서치 필요 여부 판단

### 5.2 단계 2: 리서치 수행

브랜드, 제품, 시장, 타겟, 경쟁사, 사용 맥락, 최근 캠페인, 광고 소재 아이디어 등에 대한 정보가 부족한 경우 External Research / Web Search Agent가 검색과 요약을 수행해야 한다.

리서치 목적은 다음과 같다.

1. 광고 대상 이해 보강
2. 제품 USP 및 차별점 파악
3. 경쟁사 및 유사 카테고리 광고 탐색
4. 외부 참고 레퍼런스 확보
5. 내부 레퍼런스 DB 부족 시 보완

### 5.3 단계 3: 기획 정보 보완

기획 정보가 충분하지 않을 경우 Conversation Manager Agent는 추가 질문을 수행하고, 응답을 구조화된 기획 정보에 반영해야 한다.

### 5.4 단계 4: 카피/컨셉 드래프트 생성

Copy Draft Generator Workers는 서로 다른 방향성을 가진 카피/컨셉 초안을 병렬 생성해야 한다.

### 5.5 단계 5: 드래프트 정리 및 선택 지원

Copy Curator Agent는 생성 결과를 중복 제거, 비교, 요약하여 사용자가 쉽게 선택할 수 있는 형태로 정리해야 한다.

### 5.6 단계 6: 선택 / 조합 / 재생성

사용자가 특정 드래프트를 선택하거나 둘 이상의 안을 조합해 달라고 요청할 경우 Copy Merge / Remix Agent가 새 초안을 생성해야 한다.

### 5.7 단계 7: 컨셉 확정

Concept Finalizer Agent는 최종 선택 결과를 바탕으로 컨셉, 핵심 메시지, 카피 방향, 표현 규칙을 정리해야 한다.

### 5.8 단계 8: 시나리오 작성

Scenario Writer Agent는 확정된 컨셉과 카피 방향을 기반으로 실제 광고 시나리오를 작성해야 한다.

### 5.9 단계 9: 컷 분해

Shot Planning Agent는 시나리오를 컷 단위로 분해하고 shot list를 생성해야 한다.

### 5.10 단계 10: 컷별 이미지 프롬프트 생성

Image Prompt Generation Agent는 각 컷에 대응하는 이미지 생성 프롬프트를 생성해야 한다.

### 5.11 단계 11: 이미지 생성

외부 이미지 생성 모델 또는 내부 이미지 생성 모듈은 컷별 이미지를 생성해야 한다.

### 5.12 단계 12: 콘티 구성

Storyboard Composition Agent는 시나리오, 컷 정보, 이미지, 자막/대사를 바탕으로 콘티를 구성해야 한다.

### 5.13 단계 13: 검수 및 저장

Review / QA Agent는 산출물을 검수해야 하며, Memory / Context Agent와 Diff Tracker Agent는 전체 결과와 이력을 저장해야 한다.

## 6. 에이전트별 요구사항

### 6.1 Conversation Manager Agent

역할:
사용자와 직접 대화하는 메인 에이전트이자 전체 흐름 제어자

주요 요구사항:

1. 현재 작업 단계와 세션 상태를 관리해야 한다.
2. 부족한 기획 정보와 리서치 필요 여부를 판단해야 한다.
3. 사용자 선택 및 수정 요청을 구조화된 작업으로 분해해야 한다.
4. 하위 에이전트 결과를 사용자에게 이해하기 쉬운 형태로 통합 전달해야 한다.
5. "A와 B를 합쳐달라"와 같은 복합 요청을 실행 가능한 작업 단위로 변환해야 한다.

### 6.2 기획 정보 정리 에이전트

역할:
사용자 발화를 광고 제작에 필요한 구조화된 기획 정보로 변환

주요 요구사항:

1. 제품/서비스, 타겟, 목적, 매체, 톤, 필수 요소, 금지 요소, 원하는 방향을 구조화해야 한다.
2. 누락된 정보를 식별해야 한다.
3. 명시된 정보, 추론 정보, 불확실한 정보를 구분해야 한다.
4. 이후 에이전트들이 공통 참조할 수 있는 작업 기준을 제공해야 한다.
5. 외부 리서치 결과를 반영해 기획 정보를 업데이트할 수 있어야 한다.

### 6.3 Memory / Context Agent

역할:
프로젝트/감독/브랜드 단위의 문맥 저장 및 조회

주요 요구사항:

1. 감독별 톤 선호를 저장해야 한다.
2. 브랜드별 금지 표현과 선호 표현을 저장해야 한다.
3. 과거 선택된 카피 유형과 자주 폐기되는 패턴을 저장해야 한다.
4. 이전 시나리오 및 콘티 작업 내역을 저장해야 한다.
5. 현재 요청과 유사한 과거 세션을 조회할 수 있어야 한다.
6. 외부 리서치 결과와 내부 작업 결과를 연결 가능한 문맥으로 저장해야 한다.
7. 계정별, 프로젝트별, 브랜드별 스코프를 분리해야 한다.
8. 최신성과 중요도를 고려한 우선순위 기반 retrieval을 지원해야 한다.
9. retrieval-friendly한 저장 구조를 제공해야 한다.

### 6.4 Diff Tracker Agent

역할:
각 단계 산출물 간 차이를 추적 및 저장

주요 요구사항:

1. 초안, 선택안, 조합안, 최종안 간 차이를 기록해야 한다.
2. 반복되는 수정 패턴을 식별해야 한다.
3. 특정 안의 채택/폐기 이유를 저장해야 한다.
4. 향후 추천 및 자동 조정 규칙의 근거 데이터를 제공해야 한다.

### 6.5 Internal Reference DB Search Agent

역할:
내부 광고 레퍼런스 DB 탐색 및 차용 요소 추출

주요 요구사항:

1. 업종, 톤, 포맷, 감정선, 광고 목적, 연출 스타일 기준 검색을 지원해야 한다.
2. 감독이 사전에 선별한 광고 영상 및 사례를 우선 활용해야 한다.
3. 레퍼런스 원문보다 차용 가능한 구조적 요소를 추출해야 한다.
4. 내부 DB만으로 충분하지 않은 경우 외부 리서치 단계와 연결되어야 한다.
5. embedding 검색과 metadata 필터의 조합을 지원하는 구조를 권장한다.
6. 검색 결과는 "무엇이 있는가"보다 "무엇을 차용할 수 있는가" 중심으로 후처리되어야 한다.

### 6.6 External Research / Web Search Agent

역할:
외부 웹 검색을 통해 브랜드, 제품, 시장, 경쟁사, 사용자 맥락, 외부 레퍼런스를 조사

주요 요구사항:

1. 브랜드/제품 기본 정보를 조사해야 한다.
2. USP, 포지셔닝, 특징, 최근 메시지를 탐색해야 한다.
3. 경쟁사 또는 유사 서비스 조사를 수행해야 한다.
4. 현재 작업에 필요한 배경 맥락을 확보해야 한다.
5. 외부 광고 사례 및 캠페인을 탐색해야 한다.
6. 내부 레퍼런스 DB가 부족할 때 보완 자료를 제공해야 한다.
7. 검색 결과를 단순 링크 나열이 아니라 작업 가능한 인사이트로 정리해야 한다.
8. 검색 트리거 조건을 정의해야 한다.
9. 검색 결과 신뢰도를 평가해야 한다.
10. 검색 내용을 생성 입력으로 사용하기 전 요약/정제 단계를 거쳐야 한다.
11. 내부 레퍼런스 DB 검색과 상호 보완적으로 동작해야 한다.

### 6.7 Copy Draft Generator Workers

역할:
다양한 방향의 카피/컨셉 초안을 병렬 생성

주요 요구사항:

1. 서로 다른 각도의 드래프트를 생성해야 한다.
2. 초안 간 다양성을 확보해야 한다.
3. 기획 정보, Memory, 내부 레퍼런스, 외부 리서치 결과를 함께 반영해야 한다.
4. 10개 내외의 초안을 안정적으로 생성할 수 있어야 한다.
5. 워커별 generation profile 분리를 고려해야 한다.
6. 중복 결과 제거 메커니즘이 필요하다.

### 6.8 Copy Curator Agent

역할:
다수의 드래프트를 비교, 요약, 선별

주요 요구사항:

1. 초안 간 중복을 분석해야 한다.
2. 각 초안의 강점을 정리해야 한다.
3. 사용자가 선택하기 쉬운 구조로 결과를 정리해야 한다.
4. 후속 선택 및 조합 요청을 돕는 비교 관점을 제공해야 한다.

### 6.9 Copy Merge / Remix Agent

역할:
선택된 드래프트의 장점을 조합하여 새 초안을 생성

주요 요구사항:

1. 선택안 간 조합을 지원해야 한다.
2. 특정 부분만 차용한 재구성을 지원해야 한다.
3. 톤 변형을 지원해야 한다.
4. 특정 초안 기반 확장 생성을 지원해야 한다.

### 6.10 Concept Finalizer Agent

역할:
최종 컨셉과 카피 방향 확정

주요 요구사항:

1. 선택 및 조합 결과를 정리해야 한다.
2. 시나리오 작성 단계에서 유지해야 할 핵심 방향을 고정해야 한다.
3. 광고 메시지의 중심축과 표현 규칙을 명확히 정리해야 한다.

### 6.11 Scenario Writer Agent

역할:
최종 컨셉과 카피를 광고 시나리오로 전개

주요 요구사항:

1. 장면 흐름을 설계해야 한다.
2. 대사, 내레이션, 자막을 구성해야 한다.
3. 러닝타임과 매체 특성을 반영해야 한다.
4. 내부/외부 레퍼런스에서 확보한 연출 요소를 반영해야 한다.

### 6.12 Shot Planning Agent

역할:
시나리오를 컷 단위로 분해하고 shot list를 생성

주요 요구사항:

1. 컷을 분리해야 한다.
2. 컷의 목적을 정의해야 한다.
3. shot type 및 화면 중심 포인트를 정의해야 한다.
4. 시간 배분 초안을 설계해야 한다.
5. 시나리오를 시각 제작 가능한 구조로 변환해야 한다.

### 6.13 Image Prompt Generation Agent

역할:
각 컷의 이미지 생성 프롬프트 생성

주요 요구사항:

1. 컷 목적에 맞는 피사체, 행동, 배경, 구도, 조명, 무드를 정의해야 한다.
2. 컷 간 비주얼 일관성 유지 조건을 반영해야 한다.
3. 브랜드 무드 및 레퍼런스 스타일을 반영해야 한다.
4. 이미지 생성 모델에서 바로 활용 가능한 수준으로 시각 요구사항을 정리해야 한다.
5. 컷별 프롬프트 외에 전체 콘티의 visual consistency를 관리하는 공통 조건이 필요하다.

### 6.14 Storyboard Composition Agent

역할:
시나리오, 컷 정보, 이미지, 자막/대사를 바탕으로 콘티 구성

주요 요구사항:

1. 컷 순서를 배치해야 한다.
2. 콘티 레이아웃을 구성해야 한다.
3. 컷별 설명과 텍스트를 결합해야 한다.
4. 감독 검토용 시각 보드를 생성해야 한다.
5. 이후 수정 가능한 형태의 결과물을 제공해야 한다.
6. 초기 버전은 웹 UI 기반 카드형/그리드형 보드를 지원해야 한다.
7. 이후 export 기능 확장이 가능해야 한다.

### 6.15 Review / QA Agent

역할:
카피, 시나리오, 콘티 품질 점검

주요 요구사항:

1. 기획 정보 충족 여부를 점검해야 한다.
2. 메시지 일관성을 점검해야 한다.
3. 브랜드 적합성을 점검해야 한다.
4. 과장 표현 및 정책 위반 가능성을 점검해야 한다.
5. 컷 흐름과 콘티 정합성을 검토해야 한다.
6. 외부 레퍼런스 차용이 과도하거나 부적절하지 않은지 검토해야 한다.

## 8. 상태 관리 요구사항

### 8.1 게이트 기반 상위 상태

사용자 관점의 세션 상태는 게이트를 기준으로 다음 5단계로 구분된다.

| 상태 | 설명 | 게이트 |
|------|------|--------|
| `pre_gate1` | 기획 수집 + 리서치 + 드래프트 생성 중 | — |
| `at_gate1` | 드래프트 선택 대기 중 | GATE 1 |
| `pre_gate2` | 컨셉 확정 + 리믹스 중 | — |
| `at_gate2` | 컨셉 승인 대기 중 | GATE 2 |
| `pre_gate3` | 시나리오 작성 + 컷 분해 + 프롬프트 생성 중 | — |
| `at_gate3` | 시나리오/컷 승인 대기 중 | GATE 3 |
| `pre_gate4` | 이미지 생성 + 콘티 구성 + QA 검수 중 | — |
| `at_gate4` | 최종 승인 대기 중 | GATE 4 |
| `completed` | 최종 승인 완료 | — |

### 8.2 내부 에이전트 상태

오케스트레이터가 내부적으로 관리하는 세부 상태는 다음과 같다.

1. `collecting_info`
2. `researching`
3. `copy_generation`
4. `copy_selection`
5. `concept_finalized`
6. `scenario_written`
7. `shot_planned`
8. `prompt_generated`
9. `storyboard_composed`
10. `review_completed`

이 상태들은 UI에 직접 노출되지 않으며, 오케스트레이터가 에이전트 호출 순서와 병렬 처리를 결정하는 기준으로 사용된다.

### 8.3 세션 지속성

- **타임아웃 없음**: 세션은 사용자가 명시적으로 종료하거나 새 프로젝트를 시작할 때까지 유지된다. 광고 기획은 며칠에 걸쳐 진행될 수 있으므로 강제 만료를 적용하지 않는다.
- **세션 복구**: 사용자가 돌아왔을 때 마지막 게이트 상태와 산출물이 복원되어야 한다.

### 8.4 게이트 롤백

- 사용자는 채팅으로 이전 게이트 복귀를 요청할 수 있다. ("드래프트 선택으로 돌아가고 싶어")
- 오케스트레이터는 해당 게이트로 상태를 복귀하고, 이후 단계 산출물을 새로 생성한다.
- 이전 산출물은 삭제되지 않고 Diff 이력에 보존된다.

### 8.5 상태별 유지 데이터

상태 관리에서는 다음 정보를 반드시 유지해야 한다.

1. 현재 게이트 상태 (상위 상태)
2. 현재 에이전트 실행 상태 (내부 상태)
3. 현재 확정된 기획 정보
4. 리서치 결과 반영 여부
5. 선택된 드래프트
6. 최종 컨셉
7. 시나리오
8. 컷 정보
9. 이미지 프롬프트
10. 콘티 결과
11. 검수 상태
12. 게이트별 승인 이력

## 10. 비기능 요구사항

1. 시스템은 멀티에이전트 병렬 처리 구조를 지원해야 한다.
2. 각 에이전트의 결과는 재사용 가능하고 추적 가능한 구조화 데이터로 반환되어야 한다.
3. 단계 간 산출물은 후속 단계에서 손실 없이 참조 가능해야 한다.
4. 리서치 결과, 레퍼런스 결과, 생성 결과는 출처 및 생성 근거와 함께 저장되는 것이 바람직하다.
5. 사용자 선택 및 수정 흐름은 재현 가능해야 하며, 이후 추천 로직 개선에 활용 가능해야 한다.
6. 시스템은 향후 콘티 export, 이미지 생성 모델 교체, 내부 DB 확장에 대응 가능한 모듈형 구조를 가져야 한다.

## 11. 구현 우선순위 제안

### 11.1 MVP 범위

MVP는 1인 세션 기준으로 설계하며, 다음 순서로 구현하는 것을 권장한다.

1. Conversation Manager, 기획 정보 정리, 게이트 기반 상태 관리
2. Memory / Context (읽기 전용 retrieval), Diff Tracker
3. Internal Reference Search, External Research
4. Copy Draft Generator Workers, Copy Curator, Copy Remix
5. Concept Finalizer, Scenario Writer, Shot Planning
6. Image Prompt Generation
7. Storyboard Composition (웹 UI 카드형)
8. Review / QA 및 정책 검수

**MVP 포함 UI 요소:**
- Feed 기반 채팅 인터페이스 (Chat-first)
- 에이전트 한 줄 상태 메시지
- 드래프트 카드, 컨셉 블록, 컷 스트립, 콘티 썸네일
- 4개 게이트의 Approve Bar (피드 하단 고정)
- 게이트 롤백 (채팅 요청)

**MVP 제외 항목:** 섹션 12 참조

## 12. v2 범위 (MVP 제외 항목)

다음 항목은 MVP 완성 이후 v2에서 구현한다.

| 항목 | 설명 |
|------|------|
| 협업 모드 (다중 사용자) | 동일 세션에 여러 사용자가 참여하는 실시간 협업. MVP는 1인 세션 기준. |

| 이미지 생성 고도화 | 모델 교체, 스타일 제어, 재생성 비교, 컷별 일관성 강화 등. |

| 수정 패턴 기반 개인화 | Diff 이력을 활용한 감독/브랜드별 자동 조정 규칙 학습. |
