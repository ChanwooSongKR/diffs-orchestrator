# Mockup Manual Checklist

- Open `mockup/index.html` in a browser.
- Confirm the page keeps the dark studio look and the workspace reads as chat-centered, not dashboard-first.
- Create a project with only a name and confirm it appears as a lightweight launcher card.
- Click a project card and confirm the workspace opens immediately.
- Confirm the left rail shows task groups such as Research, Drafts, Concept, Scenario, Storyboard, and QA.
- Confirm each task row shows a task status, not an agent name.
- Click a task row and confirm the task detail popup opens.
- Confirm the popup can be closed from the Close button.
- Confirm the popup can be closed from the backdrop.
- Confirm the popup shows task reason, internal agents used, recent outputs, and the next required action.
- Confirm the chat feed renders distinct user messages, system messages, status blocks, and result cards.
- Confirm user messages align right and system messages align left.
- Confirm status blocks read differently from normal chat bubbles.
- Confirm result cards are visually separate from plain text messages.
- Confirm the right focus panel stays visible beside the centered chat feed.
- Confirm the approve bar appears only when approval context is present and stays separate from the composer.
